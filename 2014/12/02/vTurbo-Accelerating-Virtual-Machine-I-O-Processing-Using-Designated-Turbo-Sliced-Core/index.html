vTurbo: Accelerating Virtual Machine I/O Processing Using Designated Turbo-Sliced Core

  • Posted on: Dec 2 2014
  • By: Ryousei Takano
  • With:

Title: vTurbo: Accelerating Virtual Machine I/O Processing Using Designated Turbo-Sliced Core
Authors: Cong Xu, Sahan Gamage, Hui Lu, Ramana Kompella, and Dongyan Xu, Purdue University
2013 USENIX ATC [URL]
Reference:

  • Improving VM Network I/O Performance (Purdue Univ.)
    • vSnoop, vFlood, vSlicer, vPipe, …
  • The authors present vTurbo, a system that offloads I/O processing for VMs to a designated core with small scheduling time-slice. It enables us to reduce both VM scheduling latency and low I/O IRQ latency.
  • This system divides CPU cores into regular cores (normal CPU time-slice, e.g., 30ms) and turbo cores (small CPU time-slice, e.g., 0.1ms). I/O processing for VMs is assigned to one or more turbo core(s). A turbo vCPU of each VM is pinned to the corresponding turbo core.
  • The prototype is implemented based on Xen 4.1.2.
  • For improving TCP performance, it uses a technique similar to their previous work, vSnoop [SC2010], that offloads ACK processing from a guest OS to the hypervisor. Such a hypervisor intervention is similar to this work.
  • The problem statement is similar to ELVIS [USENIX2013], but ELVIS uses polling instead of thread scheduling.
