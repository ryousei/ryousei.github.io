IX: A Protected Dataplane Operating System for High Throughput and Low Latency

  • Posted on: Jan 15 2015
  • By: Ryousei Takano
  • With:

Title: IX: A Protected Dataplane Operating System for High Throughput and Low Latency
Authors: Adam Belay, Stanford University; George Prekas, École Polytechnique Fédérale de Lausanne (EPFL); Ana Klimovic, Samuel Grossman, and Christos Kozyrakis, Stanford University; Edouard Bugnion, École Polytechnique Fédérale de Lausanne (EPFL)
2014 USENIX OSDI [URL] (Best paper award)

  • The authors present IX, a data plane library operating system, that provides high throughput and low latency, while maintaining strong protection offered by existing kernels.
  • IX and Arrakis [OSDI2014] propose a similar idea for a design of high performance operating systems. The both OSs use hardware virtualization to separate the data plane (network processing) from the control plane (kernel). Unlike Arrakis, IX uses a full Linux kernel as the control plane.
  • The authors use the Dune module (their previous work) [OSDI2012] within Linux to enable data planes to run as application-specific OSs in VMX non-root ring 0. The IX data plane mainly consists of the Intel NIC device driver (derived from DPDK), lwIP TCP/IP stack, and the IX kernel (based on Dune). It provides a native, zero-copy API that explicitly exposes flow control to applications.
  • IX outperforms Linux and mTCP [NSDI2014].

Arrakis: The Operating System is the Control Plane

  • Posted on: Jan 14 2015
  • By: Ryousei Takano
  • With:

Title: Arrakis: The Operating System is the Control Plane
Authors: Simon Peter, Jialin Li, Irene Zhang, Dan R. K. Ports, Doug Woos, Arvind Krishnamurthy, and Thomas Anderson, University of Washington; Timothy Roscoe, ETH Zürich
2014 USENIX OSDI [URL] (Best paper award)

  • The authors present Arrakis, a new operating system, which can deliver performance close to hardware, for data centers.
  • Arrakis splits an operating system into the data plane and the control plane. Arrakis removes the kernel from the I/O data path without compromising process isolation. Arrakis control plane is only in charge of access control, resource limits, and global naming. The rest of kernel features including protection, multiplexing, and I/O scheduling, move to applications and I/O devices. Arrakis uses device hardware to deliver I/O directly to a customized user-level library.
  • Although the concept is similar with 90’s library OSs, Arrakis exploits the recent hardware I/O virtualization like SR-IOV to implement it with high performance.
  • Arrakis is inspired by Dune [OSDI2012] and it is implemented based on the Barrelfish library OS [SOSP2009].
  • The source code is available from their web page.
