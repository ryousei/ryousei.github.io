Arrakis: The Operating System is the Control Plane

  • Posted on: Jan 14 2015
  • By: Ryousei Takano
  • With:

Title: Arrakis: The Operating System is the Control Plane
Authors: Simon Peter, Jialin Li, Irene Zhang, Dan R. K. Ports, Doug Woos, Arvind Krishnamurthy, and Thomas Anderson, University of Washington; Timothy Roscoe, ETH Zürich
2014 USENIX OSDI [URL] (Best paper award)

  • The authors present Arrakis, a new operating system, which can deliver performance close to hardware, for data centers.
  • Arrakis splits an operating system into the data plane and the control plane. Arrakis removes the kernel from the I/O data path without compromising process isolation. Arrakis control plane is only in charge of access control, resource limits, and global naming. The rest of kernel features including protection, multiplexing, and I/O scheduling, move to applications and I/O devices. Arrakis uses device hardware to deliver I/O directly to a customized user-level library.
  • Although the concept is similar with 90’s library OSs, Arrakis exploits the recent hardware I/O virtualization like SR-IOV to implement it with high performance.
  • Arrakis is inspired by Dune [OSDI2012] and it is implemented based on the Barrelfish library OS [SOSP2009].
  • The source code is available from their web page.

OSv - Optimizing the Operating System for Virtual Machines

  • Posted on: Dec 4 2014
  • By: Ryousei Takano
  • With:

Title: OSv - Optimizing the Operating System for Virtual Machines
Authors: Avi Kivity, Dor Laor, Glauber Costa, Pekka Enberg, Nadav Har’El, Don Marti, and Vlad Zolotarov, Cloudius Systems
2014 USENIX ATC [URL]
Reference:

  • OSv - the operating system designed for the cloud
  • The authors present OSv, a new guest operating system designed specifically for running a single application on a virtual machine in the cloud. The first author is famous as the father of the KVM hypervisor and a founder at Cloudius Systems.
  • OSv follows the library OS design, proposed by exokernel in 1990s. In OSv case, a hyperviosr replaces the role of exokernel.
  • OSv has been developed from scratch. The code is written in C++11.
    • Single address space: all application thread and kernel use the same page tables.
    • No spinlocks.
    • Thread scheduler: lock-free, preemptive, tick-less, fair, scalable, and efficient
    • Network channels: employ Van Jacobson’s net channel idea to reduce contentions between an application thread context and an interrupt context.
    • Beyond Linux API: shrinker, JVM balloon
  • It originally supports only Java applications, and now applications written in Lua, Ruby, and so on, run on OSv. Some guest OSs focus on running only a single high-level language. For example, Erlang on Xen, Mirage OS (OCaml) [ASPLOS2013], and Libra (J9 JVM).
